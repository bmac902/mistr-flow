# Sandcastle — run a batch on this project

This project is **observable from birth**: each Sandcastle run captures its Claude
Code session into your host `~/.claude/projects` store, so runs appear in the
Control Room fleet automatically (Control Room #193). These files are generated by
Control Room's New Project **Sandcastle module**; edit them to taste.

## One-time setup

1. **Add your token.** Copy the placeholder and fill in a Claude Code OAuth token
   (mint one on this host with `claude setup-token`). The real `.env` is
   gitignored and never committed — only `.env.example` ships.

   ```sh
   cp .sandcastle/.env.example .sandcastle/.env
   # then edit .sandcastle/.env and fill in the CLAUDE_CODE_OAUTH_TOKEN value
   ```

2. **Install dependencies** (adds `@ai-hero/sandcastle` + `tsx`):

   ```sh
   npm install
   ```

3. **Build the sandbox image** once. This tags the image `sandcastle:mistr-flow` — the
   exact name `.sandcastle/main.ts` runs against:

   ```sh
   npx @ai-hero/sandcastle docker build-image --image-name sandcastle:mistr-flow
   ```

## Run a batch

```sh
npm run sandcastle              # one iteration
npm run sandcastle -- -n 5      # five iterations
npm run sandcastle -- -m claude-opus-4-8   # a harder model
```
